MAPUploadManager

@interface MAPUploadManager

The MAPUploadManager class handles both image processing and uploading.

Properties
  • delegate

    The delegate of the upload manager object.

  • testUpload

    Set this property to YES if you want to test your upload. It uploads to a test server instead of the production server. Images on this server are not accessible and will be deleted.

    Default is NO.

  • deleteAfterUpload

    Set this property to NO if you want to keep your local images efter they have been uploaded. This flag is only used if testUpload is set to YES. When uploading to the production servers images are always deleted to avoid duplicates.

    Default is YES.

  • allowsCellularAccess

    Set this flag to YES if you want to allow uploads via cellular network.

    Default is YES.

  • numberOfSimultaneousUploads

    Controls the number of files that are uploaded simultaneously. Set this before starting an upload.

    Default is 4.

Creating an Upload Manager
  • +sharedManager

    Returns the shared upload manager object.

    Return Value

    The shared upload manager.

Starting and stopping image processing
  • -processSequences:forceReprocessing:

    Starts to process the sequences in the array.

    Parameters

    forceReprocessing

    If set to YES, images that are already processed will be processed again. This is needed if you change GPX data, like the offset etc. If set to NO, images that already have been processed will not be processed again.
  • -stopProcessing

    Stops the image processing.

Starting and stopping image uploading
  • -processAndUploadSequences:forceReprocessing:

    Starts to process and upload the sequences in the array. As soon as one image has finished processing, it will be scheduled for uploading.

    Parameters

    forceReprocessing

    If set to YES, images that are already processed will be processed again. This is needed if you change GPX data, like the offset etc. If set to NO, images that already have been processed will not be processed again.
  • -uploadSequences:

    Starts to upload the sequences in the array. If the images haven’t been processed using processSequences, they will be processed before uploading.

  • -stopUpload

    Stops the image processing and upload.

Getting the current status of the Upload Manager
  • -getStatus

    Returns the current status of the Upload Manager.

    Return Value

    The current status of the Upload Manager.
