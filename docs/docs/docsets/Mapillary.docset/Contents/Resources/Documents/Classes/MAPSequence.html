MAPSequence

@interface MAPSequence

The MAPSequence class describes a sequence. A sequence consists of images and locations. Internally, all data is stored in a compliant GPX file which works in other programs as well if you decide to export it.

The location of an image is calculated using the date and time the image was captured and is interpolated between GPS points in the GPX file. processImage If you want to export the image and get all the EXIF data added, you need to process the image first using -[MAPSequence processImage:].

Properties
  • directionOffset

    The direction offset in degrees. -1 means not set, 0 means straight forward, 90 means 90 degress to the right etc. When the heading is calculated it uses this value to correct the heading.

    If the value is -1 the compass angle will be used. If the value is 0, the compass angle is ignored and instead the value is interpolated between two locations. If the value is >0, the compass angle is ignored and instead the value is interpolated between two locations and also adds this value.

  • timeOffset

    The time offset in milliseconds.

    Default is nil. It means it is not set. +1 means the image date is increased by 1 ms when calculating the position of and image. -1 means the image date is decreased by 1 ms when calculating the position of and image.

  • sequenceDate

    The date the sequence was captured.

  • path

    The path to the directory containing this sequence.

  • gpxPath

    The path to the GPX file for this sequence. Can be nil.

  • organizationKey

    The organization this sequence belongs to. Default is nil.

  • isPrivate

    If the sequence is private or public. Only used if organizationKey is set. Default is NO.

  • sequenceKey

    The unique key of this sequence.

  • device

    The device used to capture this sequence.

  • imageCount

    Number of images in this sequence.

  • sequenceSize

    The size in bytes of the all the images in this sequence.

  • rigSequenceUUID

    The common identifier of this sequence for a rig. This is used to connect several separate sequences together.

  • rigUUID

    The unique identifier of the rig used to capture this sequence.

Initializers
  • -initWithDevice:

    Creates a new sequence for a specific device with the current date.

    Parameters

    device

    A device.
  • -initWithDevice:andDate:

    Creates a new sequence for a specific device with a specific date.

    Parameters

    device

    A device.
    date

    The date of the sequence.
  • -initWithPath:parseGpx:

    Creates and initilizes a sequence from an existing sequence.

    Parameters

    path

    The path to the directory containing the sequence.
    parseGpx

    If YES, the GPX file will be parsed and all properties will be initialized. If NO, the GPX file will not be parsed and some properties will not be initialized. Setting this parameter to YES is much slower than NO.
